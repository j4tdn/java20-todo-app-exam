2. Viết lệnh thực hiện gộp nhánh bug vào nhánh feature sử dụng git merge

git chekout feature
git merge bug 

3. Viết lệnh thực hiện gộp nhánh feature vào bug sử dụng git rebase

git chekout bug
git rebase feature

4. Phân biệt git merge và git rebase(giống, khác nhau, khi nào nên sử dụng)

giống: -đều để gộp 2 nhánh
khác:
merge: -tạo commit mới khi gộp
 	 - fix 1 lần nếu bị conflict
rebase: -không tạo commit mới khi gộp
	  - fix nhiều lần nếu bị conflict
khi nào nên sử dụng:
- gộp 2 nhanh chứa nhiều commit -> dùng merge
- tránh conflict nhiều lần -> dùng merge

5. Phân biệt git fetch và git clone

git fetch thì chỉ lấy các thay đổi từ remote về local có sẵn
git clone vừa lấy các dữ liệu từ remote về vừa tạo sẵn 1 local
 
6. Phân biệt git fetch và git pull

Git fetch khác với git pull, vì nó chỉ lấy về các thay đổi từ 
remote repository mà không thực hiện việc merge.

7. Phân biệt git pull và git pull --rebase

git pull là sự kết hợp giữa git fetch và git merge
git pull --rebase giúp hợp nhất remote branch với local branch nhanh chóng.
